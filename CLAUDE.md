# עקרונות עבודה — אפליקציות רגבים בנימין

קובץ זה מגדיר את הנחות העבודה הקבועות לכל פעולת קוד בפרויקטי רגבים בנימין
(regavim-budget, regavim-management, regavim-dorm). יש לפעול לפיהן בכל משימה, בלי צורך שהמשתמש יחזור עליהן.

## עיקרון 1: שאלות הבהרה לפני שינויים מהותיים

לפני כל **תוספת, שינוי או שדרוג מהותי** — פיצ'ר חדש, שינוי עיצוב, שינוי התנהגות קיימת, שדרוג —
יש לשאול את המשתמש כמה שיותר שאלות הבהרה (בעברית, רצוי עם כלי AskUserQuestion) כדי להבין
בדיוק למה התכוון, **לפני** שכותבים קוד. לא לנחש ולא להניח.

חריג: תיקוני באג ברורים ושינויי טקסט פשוטים — לבצע ישירות בלי שאלות.

## עיקרון 2: שפת עיצוב אחידה — לפי אפליקציית "ניהול עבודה חקלאית"

כל UX, UI ועיצוב — באפליקציות קיימות ובאפליקציות חדשות — חייב להתאים לשפת העיצוב של
אפליקציית ניהול העבודה החקלאית ("ניהול עבודה חקלאית — רגבים בנימין"). שפת העיצוב המלאה מתועדת להלן.

### בסיס

- עברית, RTL: `<html lang="he" dir="rtl">`
- פונט: `font-family: "Segoe UI", "Rubik", Arial, sans-serif;` גודל בסיס 15px
- רקע עמוד בהיר ותוכן בכרטיסים לבנים עם צל עדין
- רוחב תוכן מקסימלי: 1400px ממורכז, ריפוד 18px

### צבעים (משתני CSS — להעתיק כלשונם)

```css
:root {
  --green: #2e7d32;        /* צבע ראשי */
  --green-dark: #1b5e20;   /* ירוק כהה — כותרות, טקסט מודגש */
  --green-light: #e8f5e9;  /* ירוק בהיר — רקעי הדגשה, כותרות טבלה */
  --accent: #f9a825;       /* צהוב-ענבר — הדגשות משניות */
  --bg: #f4f6f8;           /* רקע עמוד */
  --card: #ffffff;         /* רקע כרטיס */
  --border: #d6dce1;       /* מסגרות */
  --text: #1c2733;         /* טקסט ראשי */
  --muted: #6b7884;        /* טקסט משני */
  --danger: #c62828;       /* אדום — מחיקה/שגיאה */
  --radius: 10px;
  --shadow: 0 1px 3px rgba(0,0,0,.08), 0 1px 2px rgba(0,0,0,.04);
}
```

### מבנה מסך

- **סרגל עליון (topbar)**: גרדיאנט ירוק `linear-gradient(180deg, var(--green), var(--green-dark))`,
  טקסט לבן; מכיל לוגו אימוג'י + שם האפליקציה (`h1` בגודל 18px, משקל 600), טאבים של ניווט ראשי,
  וחיווי סטטוס שמירה בצד.
- **טאבים ראשיים**: כפתורים בתוך הסרגל — רקע `rgba(255,255,255,.12)`, radius 8px;
  טאב פעיל: רקע לבן וטקסט `--green-dark` מודגש.
- **תת-טאבים (subtabs)**: כפתורי גלולה (radius 20px) עם מסגרת; פעיל: רקע ירוק וטקסט לבן.
- **כותרת עמוד (page-head)**: `h2` בגודל 20px בצבע `--green-dark`, כפתורי פעולה באותה שורה.

### קומפוננטות

- **כפתור (btn)**: רקע ירוק, טקסט לבן, radius 8px, ריפוד 8px 14px;
  וריאנטים: `secondary` (לבן + מסגרת), `accent` (ענבר), `danger` (לבן + מסגרת ואדום), `small`.
- **שדות טופס**: מסגרת `--border`, radius 7px; בפוקוס: `outline: 2px solid var(--green-light)`
  ומסגרת ירוקה. תוויות (`label`) קטנות (13px) בצבע `--muted` מעל השדה.
- **טבלאות (table.grid)**: רקע לבן, radius 10px, צל; כותרות עם רקע `--green-light`
  וטקסט `--green-dark`; יישור לימין; שורה בריחוף מקבלת רקע `#fafdfb`.
- **כרטיסים (card)**: רקע לבן, radius 10px, צל עדין, ריפוד 14px.
  כרטיס נושא יכול לקבל פס עליון: `border-top: 4px solid var(--green)`.
- **צ'יפים (chip/tag)**: רקע `--green-light`, מסגרת ירוקה, טקסט `--green-dark`, radius 16px.
- **כרטיסי סיכום (totbar)**: כרטיסים קטנים עם מספר גדול (20px, `--green-dark`) ותווית קטנה (`--muted`).
- **מודאל**: רקע כהה שקוף, חלון לבן `width: min(560px, 92vw)`, כותרת/גוף/תחתית מופרדים בקו.

### הדפסה

לכל מסך שרלוונטי לו פלט מודפס: `@media print` שמסתיר ניווט וכפתורים (`.no-print`),
מבטל צללים, ומחליף למסגרות `1px solid #999`. מחלקת `.print-only` לתוכן ייעודי להדפסה.

### כשמעדכנים אפליקציה קיימת שעיצובה שונה

לא להחליף את כל העיצוב על דעת עצמך — ליישר רכיבים חדשים לשפה הזו, ולהציע למשתמש
(בהתאם לעיקרון 1) האם ומתי ליישר את המסכים הקיימים.

## הקשר כללי

- האפליקציות משרתות את ישיבת "רגבים בנימין"; קהל היעד אינו טכני.
- ממשק המשתמש וכל הטקסטים למשתמש — בעברית. תקשורת עם המשתמש — בעברית.
- הודעות commit יכולות להיות בעברית או באנגלית, אך תמציתיות וברורות.
