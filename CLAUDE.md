# עקרונות עבודה — אפליקציות רגבים בנימין

קובץ זה מגדיר את הנחות העבודה הקבועות לכל פעולת קוד בפרויקטי רגבים בנימין
(regavim-budget, regavim-agriculture, regavim-dorm). יש לפעול לפיהן בכל משימה, בלי צורך שהמשתמש יחזור עליהן.

## עיקרון 1: שאלות הבהרה לפני שינויים מהותיים

לפני כל **תוספת, שינוי או שדרוג מהותי** — פיצ'ר חדש, שינוי עיצוב, שינוי התנהגות קיימת, שדרוג —
יש לשאול את המשתמש כמה שיותר שאלות הבהרה (בעברית, רצוי עם כלי AskUserQuestion) כדי להבין
בדיוק למה התכוון, **לפני** שכותבים קוד. לא לנחש ולא להניח.

חריג: תיקוני באג ברורים ושינויי טקסט פשוטים — לבצע ישירות בלי שאלות.

## עיקרון 2: שפת עיצוב אחידה — לפי אפליקציית "ניהול עבודה חקלאית"

כל UX, UI ועיצוב — באפליקציות קיימות ובאפליקציות חדשות — חייב להתאים לעקרונות העיצוב של
אפליקציית ניהול העבודה החקלאית ("ניהול עבודה חקלאית — רגבים בנימין"), כפי שמתועדים להלן.

**חשוב — צבעים לעומת עקרונות:** העקרונות המבניים (RTL, פונט, מבנה מסך, קומפוננטות, הדפסה)
**מחייבים תמיד**. פלטת הצבעים לעומת זאת **גמישה**: לכל אפליקציה יכולה להיות פלטה משלה.
הפלטה הירוקה שלהלן היא של האפליקציה החקלאית ומשמשת ברירת מחדל; באפליקציה חדשה או קיימת
עם זהות צבעונית אחרת — לשאול את המשתמש אילו צבעים הוא רוצה (עיקרון 1), ולמלא איתם את
אותם תפקידי צבע בדיוק.

### בסיס

- עברית, RTL: `<html lang="he" dir="rtl">`
- פונט: `font-family: "Segoe UI", "Rubik", Arial, sans-serif;` גודל בסיס 15px
- רקע עמוד בהיר ותוכן בכרטיסים לבנים עם צל עדין
- רוחב תוכן מקסימלי: 1400px ממורכז, ריפוד 18px

### תפקידי צבע (משתני CSS)

כל אפליקציה מגדירה את אותם משתנים עם אותם תפקידים; הערכים יכולים להשתנות מאפליקציה לאפליקציה.
הערכים שלהלן הם פלטת ברירת המחדל (של האפליקציה החקלאית):

```css
:root {
  --primary: #2e7d32;        /* צבע ראשי — כפתורים, טאב פעיל, הדגשות */
  --primary-dark: #1b5e20;   /* גוון כהה של הראשי — כותרות, טקסט מודגש */
  --primary-light: #e8f5e9;  /* גוון בהיר של הראשי — רקעי הדגשה, כותרות טבלה */
  --accent: #f9a825;         /* צבע משני מנוגד — הדגשות משניות */
  --bg: #f4f6f8;             /* רקע עמוד */
  --card: #ffffff;           /* רקע כרטיס */
  --border: #d6dce1;         /* מסגרות */
  --text: #1c2733;           /* טקסט ראשי */
  --muted: #6b7884;          /* טקסט משני */
  --danger: #c62828;         /* אדום — מחיקה/שגיאה */
  --radius: 10px;
  --shadow: 0 1px 3px rgba(0,0,0,.08), 0 1px 2px rgba(0,0,0,.04);
}
```

כללי שימוש בצבע (חלים על כל פלטה):

- לצבע הראשי שלושה גוונים: רגיל, כהה (לטקסט וכותרות) ובהיר מאוד (לרקעים עדינים).
- הרקע הכללי אפור-בהיר והתוכן בכרטיסים לבנים — לא רקעים צבעוניים גדולים.
- אדום שמור לפעולות מסוכנות ושגיאות בלבד.

### מבנה מסך

- **סרגל עליון (topbar)**: גרדיאנט של הצבע הראשי `linear-gradient(180deg, var(--primary), var(--primary-dark))`,
  טקסט לבן; מכיל לוגו אימוג'י + שם האפליקציה (`h1` בגודל 18px, משקל 600), טאבים של ניווט ראשי,
  וחיווי סטטוס שמירה בצד.
- **טאבים ראשיים**: כפתורים בתוך הסרגל — רקע `rgba(255,255,255,.12)`, radius 8px;
  טאב פעיל: רקע לבן וטקסט `--primary-dark` מודגש.
- **תת-טאבים (subtabs)**: כפתורי גלולה (radius 20px) עם מסגרת; פעיל: רקע בצבע הראשי וטקסט לבן.
- **כותרת עמוד (page-head)**: `h2` בגודל 20px בצבע `--primary-dark`, כפתורי פעולה באותה שורה.

### קומפוננטות

- **כפתור (btn)**: רקע בצבע הראשי, טקסט לבן, radius 8px, ריפוד 8px 14px;
  וריאנטים: `secondary` (לבן + מסגרת), `accent` (צבע משני), `danger` (לבן + מסגרת ואדום), `small`.
- **שדות טופס**: מסגרת `--border`, radius 7px; בפוקוס: `outline: 2px solid var(--primary-light)`
  ומסגרת בצבע הראשי. תוויות (`label`) קטנות (13px) בצבע `--muted` מעל השדה.
- **טבלאות (table.grid)**: רקע לבן, radius 10px, צל; כותרות עם רקע `--primary-light`
  וטקסט `--primary-dark`; יישור לימין; שורה בריחוף מקבלת רקע בהיר עדין.
- **כרטיסים (card)**: רקע לבן, radius 10px, צל עדין, ריפוד 14px.
  כרטיס נושא יכול לקבל פס עליון: `border-top: 4px solid var(--primary)`.
- **צ'יפים (chip/tag)**: רקע `--primary-light`, מסגרת בצבע הראשי, טקסט `--primary-dark`, radius 16px.
- **כרטיסי סיכום (totbar)**: כרטיסים קטנים עם מספר גדול (20px, `--primary-dark`) ותווית קטנה (`--muted`).
- **מודאל**: רקע כהה שקוף, חלון לבן `width: min(560px, 92vw)`, כותרת/גוף/תחתית מופרדים בקו.

### הדפסה

לכל מסך שרלוונטי לו פלט מודפס: `@media print` שמסתיר ניווט וכפתורים (`.no-print`),
מבטל צללים, ומחליף למסגרות `1px solid #999`. מחלקת `.print-only` לתוכן ייעודי להדפסה.

### כשמעדכנים אפליקציה קיימת שעיצובה שונה

לא להחליף את כל העיצוב על דעת עצמך — ליישר רכיבים חדשים לשפה הזו, ולהציע למשתמש
(בהתאם לעיקרון 1) האם ומתי ליישר את המסכים הקיימים.

## הקשר כללי

- האפליקציות משרתות את ישיבת "רגבים בנימין"; קהל היעד אינו טכני.
- ממשק המשתמש וכל הטקסטים למשתמש — בעברית. תקשורת עם המשתמש — בעברית.
- הודעות commit יכולות להיות בעברית או באנגלית, אך תמציתיות וברורות.
